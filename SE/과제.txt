2. servlet과 jsp의 구조 매커니즘 원리 파악
서블릿 
- HTTP protocol 서비스를 지원하는 javax.servlet.http.HttpServlet클래스를 상속하여 개발하며, servlet은 컨테이너에 의해서 실행되고 관리된다.
JSP
-HTML 문서 안에 java 코드를 포함하고 있다. MVC에서 View 역할을 한다.
-서블릿은 자바 코드 안에 HTML을 포함하고 있다. 주로 MVC에서 컨트롤러 역할을 한다.

3. collections 실생활 예
(1) list(중복저장 가능) : 학생 명단, 시험 성적, 버킷 리스트
(2) set(중복저장 불가능, 순서대로 저장 보장 x) : 로또번호
(3) map (키벨류 형식의 저장방식) : 병원 데이터 조회(주민번호)

4. file 입출력
FileReader 
- 문자 기반 스트림
- 텍스트 파일을 프로그램으로 읽어들일 때 사용
- 문자 단위로 읽으므로 텍스트만 읽기 가능

FileWriter
- 문자 기반 스트림으로 텍스트 데이터를 파일에 저장할 때 사용
- 문자 단위로 저장하므로 텍스트만 저장 가능

InputStream
- 바이트 기반 입력 스트림의 최상위 클래스로 추상클래스
- 모든 바이트 기반 입력 스트림은 이 클래스를 상속받아서 만들어짐
- InputStream 클래스에는 바이트 기반 입력 스트림이 기본적으로 가져야 할 메소드들이 정의되어 있음

OutputStream
- 바이트 기반 출력 스트림의 최상위 클래스로 추상클래스
- 모든 바이트 기반 출력 스트림 클래스는 이 클래스를 상속받아서 만들어짐
- OutputStream 클래스에는 모든 바이트 기반 출력 스트림이 기본적으로 가져야할 메소드가 정의 되어 있음

바이트 단위로 입출력하는 Stream
2바이트인 문자단위로 입출력하는 Reader,Writer
